What We're Building

A fully automated monthly project reporting pipeline. Today, someone manually reads 5 stream reports, writes a consolidated executive summary, compares it to last month, formats it as a document, saves it to SharePoint, and chases people for input. We're going to automate all of that.

By the end of this workshop you will have an agent that:

  1. Checks that all stream reports have been uploaded
  2. Reads and analyses all 5 reports using AI
  3. Generates a consolidated executive summary with comparison to last month
  4. Saves the report as a Word document in SharePoint
  5. Emails stakeholders with the report link
  6. Creates follow-up tasks in Planner for open action items

2. Create the AI Agent

📘 Context
This is the centrepiece of the entire solution. The agent replaces the human analyst who would normally spend hours reading 5 separate stream reports, writing a consolidated summary, and comparing it to last month's report. The agent does this in seconds using AI — and can answer follow-up questions about the content afterwards.

Create & Configure the Agent

  1. Open Copilot Studio

    Go to copilotstudio.microsoft.com.

  2. Create a new agent

    Click + Create → New agent.

  3. Fill in agent details
    FieldValue
    NameMonthly Reports Generator
    DescriptionAnalyzes monthly project reports across delivery streams, summarizes the current state, and highlights key changes, risks, and escalations compared to the previous report.

Add Instructions

In the Instructions field, paste the following. This is the "brain" — it tells the AI exactly how to analyse the reports and what format to produce:

ROLE
You are a project reporting analysis agent for a multi-stream application development program.

GOAL
Summarize the current reporting period across all provided documents into a consolidated report, then compare that report to the previous period's report to identify key changes.

CONTEXT
The project is organized into delivery streams:
- UX
- Frontend Development
- Backend Development
- Integrations
- Testing

Each stream report follows a structured format including:
- Status
- Top Risks
- Blockers
- Leadership Asks
- Delayed Tasks
- Summary / Outlook

You will be given:
1) Multiple documents for the current reporting period (one per stream)
2) One consolidated report from the previous reporting period

TASKS
1. Summarization
   - Create a concise consolidated report for the current period.
   - Preserve stream separation.
   - Capture only material information (no verbatim copying).
   - Normalize terminology so items can be compared across months.

2. Comparison
   Compare the current consolidated report against the previous report and identify key changes, focusing on:
   - New risks, blockers, leadership asks, or delayed tasks
   - Removed or resolved items
   - Material changes to existing items (scope, severity, ownership, or timeline)
   - Status changes (e.g., GREEN → AMBER, AT RISK → MISSED)
   - Escalations and de-escalations
   - New dependencies or critical-path impacts

3. Action Item Extraction
   After generating the report and comparison, identify concrete follow-up items that require attention:
   - Unresolved blockers that need escalation
   - New or escalated leadership asks
   - High-severity risks without a mitigation plan
   - Missed deadlines or critical-path items
   For each action item, provide:
   - A clear, concise task title (suitable for a Planner task)
   - A brief description explaining the context and urgency

COMPARISON RULES
- Treat items as "the same" if they refer to the same underlying issue, even if wording differs.
- Do not invent changes — if an item cannot be confidently matched, mark it as "New".
- Ignore cosmetic or editorial differences.
- Focus on leadership-relevant change, not minor delivery detail.

OUTPUT FORMAT
1. Executive Summary (max 6 bullets)
   - The most important changes since the previous report
   - What worsened, what improved, and what is newly critical

2. Change Log by Stream
   For each stream:
   - New Items
   - Modified Items
   - Resolved / Removed Items

   Use this structure:
   - Item
   - Change Type (New / Modified / Resolved)
   - Previous State
   - Current State
   - Impact / Why It Matters

3. Cross-Stream Observations
   - Changes that affect multiple streams
   - Emerging systemic risks or bottlenecks
   - Downstream impacts (e.g., UX delays affecting Frontend)

4. Open Leadership Attention
   - Net-new or escalated asks
   - Decisions or approvals now on the critical path

5. Action Items for Follow-Up
   List each action item as:
   - Task Title: [concise task name]
   - Details: [1-2 sentence description with context]

TONE AND STYLE
- Clear, factual, and concise
- Executive-friendly
- No speculation
- No recommendations unless explicitly supported by the reports

WHEN INPUT FILES ARE MISSING
- If the MBR Input folder is empty or does not contain all 5 stream reports, do NOT attempt to generate a report.
- Tell the user which reports are missing based on the expected streams: UX, Frontend Development, Backend Development, Integrations, Testing.
- If the previous period's consolidated report is missing, note that the comparison section cannot be generated but the summarization can still proceed if all 5 current stream reports are present.
💡 What's different here
Compared to the full workshop, this version includes Task 3: Action Item Extraction in the instructions. This tells the agent to identify follow-up items in a format that can be sent directly to Planner. The agent will use this when we wire up the Planner flow later.

Save the Agent

Click Save at the top. Don't publish yet — we need to add the flows and topic first.

✅ Checkpoint
Your agent has a name, description, detailed instructions (with action item extraction), and two SharePoint knowledge sources. It doesn't do anything interactive yet — that comes next.
What just happened: You created an AI agent with domain-specific expertise. The instructions act like a brief for a human analyst — they tell the AI what to look for, how to compare, and what format to produce. The knowledge sources tell it where to find the documents. Without the instructions, the AI would produce generic summaries. With them, it produces structured, executive-level output tailored to your reporting process.

3. Build the Agent Flows

📘 Context
The agent is the brain, but it can't interact with SharePoint, Outlook, or Planner on its own. You'll build 4 small flows that act as the agent's "hands":
  1. Check Input — counts files in MBR Input
  2. Save Report — creates a Word document in MBR Output
  3. Email Stakeholders — sends the report link via email
  4. Create Planner Task — creates follow-up tasks from the report
Each flow follows the same pattern: receive input from the agent → do something → respond to the agent.

Flow 1: Check Input MBR

In Copilot Studio, go to Actions and create a new Agent flow.

Flow Name: Check Input MBR (Agent)

StepWhat to SearchConfiguration
1When an agent calls the flow (trigger)No inputs needed
2SharePoint — Get files (properties only)
  • Create a SharePoint connection if you haven't already
  • Select your Site Address from the dropdown
  • Set Library Name to Documents
  • Expand Advanced parameters → in Limit Entries to Folder, browse to or type /Shared Documents/General/MBR Input
  • ⚠️ Tip: If you can't find the SharePoint action, check See more for additional connectors.
3Compose
  • Click fx → paste: length(outputs('Get_files_(properties_only)')?['body/value']) → OK
  • Rename this step to FileCount (click ... → Rename)
4Respond to the agent
  • Click + Add an output → Number
  • Name: FileCount
  • Value: click fx → paste: outputs('FileCount') → OK

Click Save.

✅ Checkpoint
Click Test → Manually → Run flow. The flow should return 6 (the 5 stream reports + 1 previous report in MBR Input).
What just happened: You built a utility flow that the agent can call on demand. The agent handles thinking; this flow handles checking. It's a pattern you'll see repeatedly — agents delegate mechanical tasks to flows.
⚠️ Troubleshooting

If the flow fails when the folder is empty, the length() expression may be receiving null. This is expected — the agent's topic will handle the "0 files" case gracefully.

Flow 2: Save MBR Report to SharePoint

Create another new Agent flow in Copilot Studio (Actions).

Flow Name: Save MBR Report to SharePoint

StepWhat to SearchConfiguration
1When an agent calls the flow (trigger)
  • Click + Add an input → Text
  • Name: ReportContent
2SharePoint — Create file
  • Site: your SharePoint site
  • Folder path: Shared Documents/General/MBR Output
  • File name: click fx → paste: concat('MBR-Report-', formatDateTime(utcNow(), 'yyyy-MM'), '-', guid(), '.md')
  • File Content: click in the field → select the ⚡ lightning bolt (dynamic content) icon → under the trigger step, select ReportContent
3Respond to the agent
  • Click + Add an output → Text
  • Name: FileLink
  • Value: click ⚡ → look under the Create file step → click See more if needed → select body/Path

Click Save.

✅ Checkpoint
You can't easily test this flow standalone (it needs text input from the agent), but verify it saved without errors. Click Test → Manually → enter any text as ReportContent → Run flow. Check that a file appears in your MBR Output folder on SharePoint.
What just happened: You built the flow that turns the agent's AI-generated text into a permanent document. The file name includes the year, month, and a unique identifier (MBR-Report-2026-04-a1b2c3d4.md) so each report is unique and kept in MBR Output as a running archive.

4. Create the "Generate Monthly Report" Topic

Topic Name: Generate Monthly Report

📘 Context
This is where everything comes together. The topic is the agent's decision tree — when a user says "Generate report", the topic orchestrates: check files → generate report → save document → email stakeholders → create tasks → return link. It's the conductor that coordinates the AI brain with all four flow-based actions.
  1. Create a new topic

    Go to Topics → + Add a topic → From blank.

  2. Set the trigger

    Keep trigger as "The agent chooses". Click Edit on the trigger → in "Describe what the topic does", paste:

    Checks if all 5 stream reports are uploaded to the MBR Input folder, generates the consolidated monthly report, saves it to SharePoint, emails stakeholders, and creates follow-up tasks in Planner.

    Click Save on the trigger.

Now build the canvas step by step:

Node 1: Check the files

Click + below the trigger → Add a tool → select Check Input MBR (Agent). This returns the FileCount variable.

Node 2: Decision

Click + → Add a node → Condition. Set: FileCount is greater than or equal to 6.

Node 3a: YES branch — Generate, Save, Email, Create Tasks

In the Yes (left) branch, add these nodes in order:

  1. Send a message

    Click + → Send a message. Text:

    All reports found. Generating your consolidated report now...
  2. Generative answers

    Click + → Advanced → Generative answers. Input:

    Create the consolidated monthly report following your instructions. Use all stream reports and the previous report from the MBR Input folder.
    • Click Edit on Data sources → make sure your MBR Input SharePoint knowledge is selected → toggle on "Search only selected sources"
    • Expand the Advanced section on the right-hand properties panel → set Save LLM response to Complete (recommended)
    • Click on Save bot response and save as the variable ReportOutput
  3. Save the report to SharePoint

    Click + → Add a tool → select Save MBR Report to SharePoint.

    For the ReportContent input, click fx (formula) and enter:

    Topic.ReportOutput.Text.Content

    This returns the FileLink variable.

  4. Email stakeholders

    Click + → Add a tool → select Email Report to Stakeholders.

    • For ReportLink: click {x} → select the FileLink variable (returned from the Save flow)
    • For ReportSummary: click fx and enter: Topic.ReportOutput.Text.Content
  5. Create Planner tasks

    Click + → Add a tool → select Create Planner Task from Report.

    • For TaskTitle: click fx and enter: Concatenate("Review MBR Action Items - ", Text(Now(), "mmmm yyyy"))
    💡 Tip — Multiple tasks
    The agent can only call this flow once in the topic canvas. For the workshop, we create one summary task. In production, you could use a Parse JSON + Apply to Each pattern in the flow to create multiple tasks from the agent's action items list. Alternatively, after the report is generated, you can ask the agent in chat: Create a Planner task for each action item — the agent will call the flow multiple times autonomously.
  6. Send confirmation message

    Click + → Send a message. Text:

    Your report has been saved to SharePoint: then click {x} and insert the FileLink variable. Continue with:

    

Stakeholders have been notified by email and a follow-up task has been created in Planner.

You can ask me follow-up questions about the report content.

Node 3b: NO branch — Tell the user

In the No (right) branch:

  1. Send a message

    Click + → Send a message. Text: Only then click {x} and insert FileCount, then continue typing: file(s) found in MBR Input. Please ensure all 5 stream reports and the previous month's report are uploaded before generating the report.

Save the topic

Click Save at the top.

Key Lessons

  1. Agents think, flows act

    Use agents when you need AI to read, understand, analyse, or write. Use flows when you need to move files, send emails, create tasks, or interact with external services.

  2. One pattern, four flows

    Every flow follows the same structure: receive input from agent → perform action → respond to agent. Once you've built one, the rest are variations on the same theme.

  3. Instructions are everything

    The quality of the agent's output depends entirely on the instructions prompt. Adding "Action Item Extraction" to the instructions is what enables the Planner integration — the agent needs to know what format to produce.

  4. The agent is interactive

    Unlike a static report, the agent can answer follow-up questions and take additional actions on demand. This makes it a tool, not just a process.
